# 用户故事框架：SERP 向用户意图发出信号

> 本文件是 `seo-agents` 的中文详细参考，已按当前项目能力边界适配。命令名、API 名、Schema 类型、JSON 字段和专用 SEO 术语按需保留英文。

从可观察的 SERP 信号中得出用户故事。每个故事都必须引用
产生它的特定信号——无需猜测。

---

## 信号源

### 1. 人们也问（PAA）

PAA 问题揭示了搜索者的**知识差距和担忧**。

- 按主题对 PAA 进行聚类：定义性（“什么是”）、程序性（“如何做”）、
  评估性（“X 值得吗”）、比较性（“X 与 Y”）、风险（“X 安全吗”）
- 每个集群映射到一个不同的角色或旅程阶段
- 定义性 PAA = 认知阶段，评估性 PAA = 考虑阶段，
  比较 PAA = 决策阶段

### 2. 广告文案

广告揭示了转化的**商业触发点和异议处理**。

- 提取所有可见广告（顶部+底部）中重复出现的主题
- 常见模式：强调免费试用（障碍：承诺）、可信计数
  （障碍：信任）、价格锚定（障碍：成本）、速度索赔（障碍：时间）
- 如果没有广告出现，则关键字可能是信息性的 - 商业意图较低

### 3.相关搜索

相关搜索揭示了**搜索旅程**——之前和之后发生的事情。

- 添加限定词的搜索（+“针对初学者”、+“定价”）= 缩小范围
- 删除限定符的搜索 = 扩大/旋转
- “[关键字]替代方案”=不满意信号
- “[关键字] vs [竞争对手]”= 主动比较阶段
- “[关键字]评论”=购买前寻求信任

### 4. 精选片段

片段格式揭示了**预期的答案结构**。

- 段落片段 = 用户想要简洁的定义或解释
- 列表片段 = 用户想要步骤或排名项目
- 表片段 = 用户想要结构化比较数据
- 视频片段=用户想要视觉演示
- 没有片段 = Google 不确定最佳格式（机会）

### 5. 人工智能概述

AI 概述揭示了 Google 认为的**权威综合**。

- 引用的来源=权威信号Google对此主题的信任
- 答案结构 = Google 认为最适合查询的格式
- 如果目标页面未被引用=内容与Google的合成模型不匹配

## 用户故事格式

```
As a [persona],
I want to [goal],
because [emotional driver],
but I'm blocked by [barrier].
```

### 角色衍生

来自信号簇，而不是假设：
- 关于成本/定价的 PAA 集群 = **注重预算的买家**
- 关于设置/实施的 PAA 集群 = **技术评估员**
- 强调信任/安全的广告 = **规避风险的决策者**
- 与“小型企业”相关的搜索 = **SMB 所有者**
- 与“企业”相关的搜索 = **企业买家**

### 情绪状态

将 SERP 信号映射到情绪状态：

|信号模式|情绪状态 |
|--------------|----------------|
| PAA 中的“X 安全/合法/诈骗吗”|持怀疑态度|
|相关 | 中的许多比较查询被选择淹没|
| “如何[基本任务]”主导PAA |困惑/沮丧|
|高广告密度+购物效果|准备购买|
| “[用例] 的最佳 [X]”结果 |仔细评估 |
| PAA 中的“[X] 不工作/问题”|对当前的解决方案感到沮丧|

### 屏障类型

|屏障| SERP 信号 |页面必须地址 |
|--------|------------|--------------------|
|信息差距|片段未回答 PAA 问题 |第一卷中清晰、完整的答案 |
|信任差距|广告强调信任徽章/评论|社会证明、证书、保证 |
|比较疲劳|许多“vs”和“best”相关搜索|明确差异化或推荐|
|价格敏感度| PAA 关于成本、免费替代品 |定价透明，价值合理|
|技术混乱 | “如何”PAA，设置相关查询 |分步指南、演示或工具 |
|时间压力|含有“即时”/“今天”/“快速”的广告 |内容快赢，解析速度快 |

## 推导示例

**关键字**：“项目管理软件”

**观察到的信号：**
- PAA：“Monday.com 比 Asana 更好吗？”、“最便宜的项目管理工具是什么？”、“小团队需要项目管理软件吗？”
- 广告：“永久免费计划”、“受到 150,000 个团队的信赖”、“5 分钟内设置”
- 相关：“小型企业项目管理软件”、“免费项目管理工具”、“Asana vs Monday vs ClickUp”
- 精选片段：带有简短描述的工具列表

**衍生的用户故事：**

1. 作为**小企业主**，我想找到负担得起的 PM 软件，因为
   我需要组织不断壮大的团队，但**价格敏感性**阻碍了我
   以及担心为我不会使用的功能付费。
   *（来源：PAA“最便宜”，相关“小型企业”，广告“永远免费”）*

2. 作为**团队领导评估工具**，我想比较前 3 个选项
   旁边，因为我的老板要求推荐，但我被阻止了
   **比较疲劳**来自太多看起来相似的选项。
   *（来源：相关“Asana vs Monday vs ClickUp”，PAA“周一比 Asana 更好”）*

3. 作为一名**非技术经理**，我想要一款无需任何操作即可立即运行的软件
   培训，因为我无法承受迁移期间的停机时间，但我被
   **关于设置复杂性的技术混乱**。
   *（来源：广告“5 分钟内设置”，PAA“小团队需要 PM 软件吗”）*

## 故事质量检查表

- [ ] 每个故事都至少引用一个特定的 SERP 信号
- [ ] 角色名称反映信号源，而不是假设
- [ ] 情感驱动因素以可观察的 SERP 基调为基础
- [ ] Barrier 足够具体，可以建议进行页面级修复
- [ ] 3-5 个故事涵盖主要意图角度（无重复）
- [ ] 故事至少跨越 2 个旅程阶段（意识、考虑、决定）

## 本项目适配边界

- 优先使用当前已实现的 `seo-agents` CLI 获取证据；没有 CLI 的能力作为 Agent playbook 或后续扩展处理。
- 不把外部数据源、付费 provider、浏览器渲染或地图 API 写成已执行，除非当前任务确实运行并取得证据。
- 每个 finding 必须有可复核 evidence；数据缺失时明确写“未配置”或“缺少输入”。
