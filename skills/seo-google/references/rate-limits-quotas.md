## 指数退避策略

当收到 429 或 5xx 错误时：

```
Attempt 1: wait 1 second
Attempt 2: wait 2 seconds
Attempt 3: wait 4 seconds
Attempt 4: wait 8 seconds
Attempt 5: wait 16 seconds
Max: give up after 5 retries
```

为每个等待添加随机抖动（0-500ms）以避免雷群。

## 常见错误代码

|代码|意义|适用于 |行动|
|------|---------|------------|--------|
| 400 |错误的请求 |全部 |检查 URL 格式、请求正文 |
| 401 | 401未经授权 |服务帐户 API |刷新凭据 |
| 403 | 403禁止| GSC、GA4、索引 |检查权限（服务帐户访问）|
| 404 | 404没有找到| CrUX、GSC |流量不足 (CrUX) 或无效属性 (GSC) |
| 429 | 429限价|全部 |后退并重试。检查 Retry-After 标头。 |
| 500 | 500服务器错误|全部 |后退重试 |
| 503 | 503服务不可用 |全部 |后退重试 |

## Retry-After 标头

某些 Google API 返回带有 429 响应的 `Retry-After` 标头。如果存在，请使用该值（以秒为单位）而不是指数退避。

## GA4 代币预算

GA4 使用令牌系统而不是简单的请求计数：
- 简单的一维、一指标报告：~1-5 个代币
- 复杂的多维度、多指标：~10-100 个代币
- 设置`returnPropertyQuota: true`来监控消耗
- 每日限额：每个项目每个房产 25,000 个代币
- 每小时限制：每个项目每个房产 5,000 个代币
- 并发：最多 10 个并发请求

## CrUX 共享配额

CrUX API 和 CrUX History API 每个项目共享相同的 150 QPM 配额。如果在同一工作流程中查询这两个 API，请相应地进行规划。

## 成本汇总

**seo-google 使用的所有 API 在正常使用水平下都是免费的**。以下情况无需计费：
- PSI、CrUX、CrUX 历史记录（API 密钥，无限免费）
- GSC（服务帐户，30M QPD）
- 索引 API（服务帐户，200 次发布/天）
- GA4 Data API（服务帐户，25K 代币/天）
- 知识图（API密钥，100K QPD）

只有自定义搜索和网络风险具有高容量付费等级。
