# 基于角色的评分方法

> 本文件是 `seo-agents` 的中文详细参考，已按当前项目能力边界适配。命令名、API 名、Schema 类型、JSON 字段和专用 SEO 术语按需保留英文。

从每个派生角色的角度对目标页面进行评分。这揭示了
页面可以很好地服务哪些受众群体，哪些受众群体则失败。

---

## 人物角色推导

从 SERP 意图信号中派生出 4-7 个角色。如果没有，就不要发明人物角色
证据——每个角色都必须追溯到一个信号簇。

### 角色衍生的来源

|来源 |它揭示了什么 |角色示例 |
|--------|----------------|-----------------|
| PAA 问题群 |观众的知识差距|初学者，技术评估员 |
|广告文案片段 |商业受众目标|预算买家、企业买家 |
|相关搜索主题 |旅程舞台组|研究员、比较购物者 |
| SERP 结果类型 |内容消费偏好 |视觉学习者、数据驱动分析师 |
|精选片段格式 |预期答案风格 |快速解答寻求者 |

### 角色卡格式

对于每个角色，记录：

```
**[Persona Name]** -- [one-line description]
- Role: [job title or situation]
- Goal: [what they want to accomplish]
- Emotional state: [from user-story-framework.md]
- Journey stage: Awareness | Consideration | Decision
- Key questions: [2-3 questions this persona needs answered]
- SERP evidence: [specific signals that generated this persona]
```

## 4 维评分标准

从 4 个维度对每个角色进行评分。每个人物最多 25 分，每个角色总共 100 分。

### 维度 1：相关性 (0-25)

该页面是否满足了该角色的特定需求？

|分数 |标准|
|--------|----------|
| 21-25 | 21-25页面通过特定内容直接解决角色的主要目标 |
| 16-20 |页面涵盖了主题，但缺乏特定角色的深度或框架 |
| 11-15 | 11-15页面是切线相关的——角色必须推断才能找到价值|
| 6-10 |页面提到了该主题，但服务于不同的受众 |
| 0-5 |页面与此角色的需求无关 |

**评分信号：**关键字与角色目标的一致性、角色相关的存在
部分（例如，针对 SMB 角色的“小型企业”部分）、覆盖深度
用于特定角色的用例。

### 维度 2：清晰度 (0-25)

这个角色能在 10 秒内找到答案吗？

|分数 |标准|
|--------|----------|
| 21-25 | 21-25对于此角色，答案在折叠上方或第一卷轴内可见 |
| 16-20 |答案在页面上，但需要滚动或导航 |
| 11-15 | 11-15答案存在，但被隐藏在密集的文本或不清晰的结构中 |
| 6-10 | Persona 必须从多个部分拼凑出答案 |
| 0-5 |答案不明确——角色会反弹 |

**评分信号：** 角色问题的标题清晰度、可扫描性（项目符号、
表格、粗体关键术语）、目录或跳转链接、信息层次结构
匹配角色优先级。

### 维度 3：信任 (0-25)

该页面是否为该角色提供了足够的信任信号？

|分数 |标准|
|--------|----------|
| 21-25 | 21-25与此角色的担忧直接相关的多个信任信号 |
| 16-20 |存在一般信任信号，但不针对特定角色 |
| 11-15 | 11-15一些信任信号，但此角色所需的东西存在差距|
| 6-10 |最小的信任信号——角色会感到不确定|
| 0-5 |没有信任信号或主动破坏此角色信任的信号 |

**评分信号：**来自类似用户/公司的推荐、相关凭证
或认证、安全徽章（针对规避风险的角色）、案例研究匹配
角色的行业或规模、作者的专业知识信号。

### 维度 4：行动 (0-25)

这个角色有明确的下一步吗？

|分数 |标准|
|--------|----------|
| 21-25 | 21-25清晰、符合人物性格、摩擦力低的 CTA |
| 16-20 | CTA 存在，但是通用的（不适合人物角色的阶段）|
| 11-15 | 11-15下一步可用，但不突出或需要搜索 |
| 6-10 |唯一一个与人物角色旅程阶段不匹配的 CTA |
| 0-5 |没有明确的下一步——人物角色进入了死胡同|

**评分信号：** CTA 与旅程阶段的相关性（意识=“了解更多”，
决策=“立即购买”）、摩擦程度（免费试用与销售电话）、CTA 展示位置
相对于角色相关内容，不同准备级别的替代路径。

## 分数解读

|范围 |评级 |寓意 |
|--------|--------|-------------|
| 80-100 |优秀|页面很好地服务了这个角色——仅进行了小的优化|
| 60-79 | 60-79好 |页面相关，但与此角色有显着差距 |
| 40-59 | 40-59需要工作|页面部分服务于这个角色——需要重大改进|
| 0-39 |严重不匹配|页面未能满足此角色 - 需要进行重大重组或新页面 |

## 聚合和优先级

### 加权平均

并非所有角色都同样重要。按估计搜索量份额划分的权重：

- 如果 SERP 显示 70% 的信息意图，则信息角色的权重更高
- 如果广告占主导地位，则商业角色的权重更高
- 如果存在本地包，则本地角色的权重更高

### 优先级排名

按以下方式对改进建议进行排序：

1. **搜索量权重最高的最弱角色** = 最大的机会
2. **所有角色中得分最低的维度**=系统性问题
3. **严重不匹配角色** = 基本页面类型问题

### 输出格式

```
## Persona Scores for [URL]

| Persona | Relevance | Clarity | Trust | Action | Total | Rating |
|---------|-----------|---------|-------|--------|-------|--------|
| [Name] | XX/25 | XX/25 | XX/25 | XX/25 | XX/100 | [Rating] |
| ... | ... | ... | ... | ... | ... | ... |

### Weakest Persona: [Name] (XX/100)
**Top issue:** [specific problem]
**Recommended fix:** [concrete action with page-level detail]

### Systemic Issues
- [dimension]: [pattern across all personas]

### Priority Actions
1. [Action targeting weakest persona]
2. [Action targeting systemic issue]
3. [Action targeting next weakest persona]
```

## 验证规则

- [ ] 每个角色都可以追溯到特定的 SERP 信号（没有发明的角色）
- [ ] 分数包括页面中的具体证据（不仅仅是数字）
- [ ] 建议是具体的（章节名称、CTA 文本、位置）
- [ ] 至少 4 个角色得分，不超过 7 个
- [ ] 在建议中首先解决最弱的角色
- [ ] 分数解释使用上述标准范围

## 本项目适配边界

- 优先使用当前已实现的 `seo-agents` CLI 获取证据；没有 CLI 的能力作为 Agent playbook 或后续扩展处理。
- 不把外部数据源、付费 provider、浏览器渲染或地图 API 写成已执行，除非当前任务确实运行并取得证据。
- 每个 finding 必须有可复核 evidence；数据缺失时明确写“未配置”或“缺少输入”。
