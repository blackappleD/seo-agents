# SEO漂移比较规则

> 本文件是 `seo-agents` 的中文详细参考，已按当前项目能力边界适配。命令名、API 名、Schema 类型、JSON 字段和专用 SEO 术语按需保留英文。

17 条规则，涵盖 3 个严重级别。每条规则都比较特定的 SEO 元素
存储的基线和当前页面状态之间。

---

## 严重（需要立即采取行动）

这些变化通常会在几天内造成明显的流量损失。

## 规则 1：架构/JSON-LD 完全删除
- **比较**：基线 `schema` 数组有项目，当前为空
- **阈值**：之前存在的任何模式，现在没有
- **操作**：立即恢复结构化数据。丰富的结果将在数小时内丢失。
- **交叉参考**：`/seo schema <url>`

## 规则 2：规范 URL 已更改
- **比较**：基线 `canonical` 与当前 `canonical`
- **阈值**：不同的非空值（标准化后）
- **操作**：验证新规范是否有意为之。不正确的规范将排名信号重定向到错误的页面。
- **交叉参考**：`/seo technical <url>`

## 规则 3：删除规范 URL
- **比较**：基线 `canonical` 已设置，当前为 `null`
- **阈值**：具有价值，但现在缺失
- **操作**：恢复规范标签。Google会猜测，对于带有查询参数的页面通常是错误的。
- **交叉参考**：`/seo technical <url>`

## 规则 4：添加 Noindex 指令
- **比较**：基线 `meta_robots` 不包含“noindex”，当前包含“noindex”
- **阈值**：现在存在“noindex”子字符串（不区分大小写）
- **操作**：如果无意，请立即删除。页面将在几天内从索引中删除。
- **交叉参考**：`/seo technical <url>`

## 规则 5：H1 标签完全删除
- **比较**：基线 `h1` 有条目，当前为空
- **阈值**：之前有一个或多个 H1，现在为零
- **操作**：恢复 H1 航向。搜索引擎的主页主题信号。
- **交叉参考**：`/seo content <url>`

## 规则 6：H1 文本发生重大变化
- **比较**：基线中的第一个 H1 与当前的第一个 H1，SequenceMatcher 比率
- **阈值**：相似率 < 0.5（>50% 不同）
- **操作**：验证 H1 更改是否与目标关键字策略保持一致。
- **交叉参考**：`/seo content <url>`

## 规则 7：title tag完全删除
- **比较**：基线 `title` 已设置，当前为 `null` 或为空
- **阈值**：具有价值，但现在缺失
- **操作**：立即恢复title tag。Google会自动生成一个，但通常效果很差。
- **交叉参考**：`/seo page <url>`

## 规则 8：HTTP 状态代码更改为错误
- **比较**：基线 `status_code` 为 2xx，当前为 4xx 或 5xx
- **阈值**：状态代码类从成功更改为客户端/服务器错误
- **操作**：调查服务器错误或丢失页面。几天之内排名就会下降。
- **交叉参考**：`/seo technical <url>`

## 警告（1 周内调查）

这些变化可能会影响排名或点击率，但有时是有意为之。

## 规则 9：标题文本已更改
- **比较**：基线 `title` 与当前 `title`（已修整）
- **阈值**：字符串不同（区分大小写，空格标准化）
- **操作**：验证新标题包含目标关键字。监控 GSC 两周内的点击率。
- **交叉参考**：`/seo page <url>`

## 规则 10：meta description已更改
- **比较**：基线 `meta_description` 与当前 `meta_description`
- **阈值**：字符串不同（已修剪）
- **操作**：验证新描述是否包含号召性用语和目标关键字。监控点击率。
- **交叉参考**：`/seo page <url>`

## 规则 11：核心网络生命指标回归 >20%
- **比较**：每个 CWV 指标 p75 值（LCP、INP、CLS）基线与当前值
- **阈值**：当前值比基线差 >20%（LCP/INP 较高，CLS 较高）
- **行动**：调查性能回归。检查最近的代码更改或第三方脚本。
- **交叉参考**：`/seo technical <url>`

## 规则 12：CWV 表现得分下降 10 分以上
- **比较**：Lighthouse 性能得分基线与当前得分
- **阈值**：下降 10 分或更多（例如，85 至 74）
- **操作**：运行完整的 PageSpeed 分析以识别新的瓶颈。
- **交叉参考**：`/seo google psi <url>`

## 规则 13：删除 OG 标签
- **比较**：基线 `open_graph` 有条目，当前为空
- **阈值**：之前有一个或多个 OG 标签，现在没有
- **操作**：恢复 OG 标签。社交共享将显示通用/缺失的预览。
- **交叉参考**：`/seo page <url>`

## 规则 14：修改架构/JSON-LD 内容
- **比较**：基线 `schema_hash` 与当前 `schema_hash`
- **阈值**：哈希值不同且模式仍然存在（删除是规则 1）
- **操作**：验证修改后的架构。检查类型更改、删除的属性或新的验证错误。
- **交叉参考**：`/seo schema <url>`

---

## 信息（仅限意识）

跟踪这些内容的完整性。通常是积极或中性的变化。

## 规则 15：添加新架构/JSON-LD
- **比较**：基线 `schema` 为空，当前有项目
- **阈值**：之前没有架构，现在存在架构
- **行动**：积极的改变。使用 `/seo schema <url>` 验证新架构。
- **交叉参考**：`/seo schema <url>`

## 规则 16：H2 结构已更改
- **比较**：基准 `h2` 阵列与当前 `h2` 阵列
- **阈值**：不同数量的 H2，或不同的 H2 文本值
- **操作**：查看标题层次结构。确保内容部分仍然与目标主题保持一致。
- **交叉参考**：`/seo content <url>`

## 规则 17：内容哈希值已更改
- **比较**：基线 `html_hash` 与当前 `html_hash`
- **阈值**：哈希值不同（涵盖任何正文内容更改）
- **操作**：检测到一般内容更改。检查是否没有触发其他规则以了解发生了什么变化。
- **交叉参考**：`/seo page <url>`

## 本项目适配边界

- 优先使用当前已实现的 `seo-agents` CLI 获取证据；没有 CLI 的能力作为 Agent playbook 或后续扩展处理。
- 不把外部数据源、付费 provider、浏览器渲染或地图 API 写成已执行，除非当前任务确实运行并取得证据。
- 每个 finding 必须有可复核 evidence；数据缺失时明确写“未配置”或“缺少输入”。
