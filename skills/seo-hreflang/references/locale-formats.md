# 国际 SEO 的 locale 格式参考

> 本文件是 `seo-agents` 的中文详细参考，已按当前项目能力边界适配。命令名、API 名、Schema 类型、JSON 字段和专用 SEO 术语按需保留英文。

用于验证内容 parity 或生成本地化内容时的格式一致性。数字、日期、货币、地址和电话格式如果与目标 locale 不一致，例如在 `de-DE` 页面使用美国日期格式，会削弱本地化可信度和用户信任。

原始概念：Chris Muller（Pro Hub Challenge）。

## 文本扩展比例

相对英文，本地化后的文本长度通常会变化。布局、按钮、导航、title tag、meta description 和结构化摘要都要预留弹性。

| 语言 | 扩展比例 | 影响 |
|---|---:|---|
| German | +25-35% | 标题、按钮、导航标签明显变长 |
| French | +15-25% | 中等扩展 |
| Spanish | +15-25% | 中等扩展 |
| Italian | +15-25% | 中等扩展 |
| Portuguese | +15-25% | 中等扩展 |
| Dutch | +10-20% | 轻微扩展 |
| Japanese | -10-25% | 收缩，更紧凑 |
| Korean | -10-20% | 收缩 |
| Chinese | -20-30% | 明显收缩 |

## 认证规则

1. 扫描页面中的数字模式，包括价格、统计数据、尺寸、评分和库存。
2. 与页面声明语言、`hreflang`、`html[lang]` 和目标市场的预期格式比较。
3. 在非美国页面上标记美国格式数字，例如 `de-DE` 页面里的 `$1,234.56`。
4. 检查博客日期、版权声明、更新日期、优惠截止日期和评论日期。
5. 验证货币符号、税费表达和运费表达是否匹配目标市场。
6. 检查电话号码是否使用国际格式，并确认国家代码正确。
7. 发现格式不一致时，不要只报“翻译问题”；要写明具体字段、页面位置、当前格式、目标 locale 和建议格式。

## 本项目适配边界

- 优先使用当前已实现的 `seo-agents hreflang <url>` 和 `seo-agents page/content <url> --json` 获取证据。
- 没有抓取到完整本地化页面时，只能报告可见页面证据和缺失输入，不推断其他 locale 已正确。
- 每个 finding 必须包含页面 URL、检测到的 locale 信号、具体格式证据和建议修正。
