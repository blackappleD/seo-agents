## 来源说明

## 概念

地理网格排名跟踪模拟来自多个 GPS 的 Google 地图搜索
围绕企业进行协调，以显示排名在不同地理区域之间的变化
区。输出是一个热图，显示企业排名靠前的位置（绿色）
以及竞争对手占主导地位的地方（红色）。

---

## 网格生成（基于半正矢）

## 算法

1、取中心坐标（营业地点）：`center_lat`、`center_lng`
2. 定义网格大小（例如，7x7 = 49 点）和半径（以公里为单位）
3.计算间距：`step = (2 * radius_km) / (grid_size - 1)`
4、利用偏移公式生成网格点：

```
For each row i (0 to grid_size-1) and column j (0 to grid_size-1):
  dy = (i - center_index) * step_km
  dx = (j - center_index) * step_km
  new_lat = center_lat + (dy / 111.32)
  new_lng = center_lng + (dx / (111.32 * cos(center_lat * pi/180)))
```

其中 `center_index = (grid_size - 1) / 2` 和 `111.32 km = 1 degree latitude`。

## 网格大小和用例

|网格|积分 |典型半径|最适合 |预计。成本（实时）|
|------|--------|----------------|----------|-----------------|
| 3x3 | 3x3 9 | 2 公里 |快速快照，低预算 | $0.018/关键字 |
| 5x5 | 5x5 25 | 25 3 公里 |规范城市审计| $0.050/关键字 |
| **7x7** | **49** | **5公里** | **默认。覆盖范围和成本的最佳平衡** | **0.098 美元/关键字** |
| 9x9 | 9x9 81 | 81 8 公里 |郊区/广阔的服务区 | $0.162/关键字 |
| 13x13 | 13x13 169 | 169 15 公里 |农村或大型地铁 | $0.338/关键字 |

**半径指南：** 城市密集= 2-5公里，郊区= 5-10公里，农村= 10-25公里。

## DataForSEO 集成

使用带有 `location_coordinate` 参数的 Google Maps SERP API：

```json
{
  "keyword": "dentist",
  "location_coordinate": "30.2672,-97.7431,15z",
  "language_code": "en",
  "device": "mobile",
  "depth": 20
}
```

对于每个网格点，使用该点的纬度/经度触发一个 API 调用。解析
`items` 数组查找目标企业排名（结果中的位置）。

**速率优化：** DataForSEO 允许每个 POST 最多执行 100 个任务。对于 7x7
grid，将所有 49 个任务批处理到单个请求中，以最大限度地减少 HTTP 开销。

## 本地语音份额 (SoLV)

指标由 Local Falcon 首创。测量整个网格的可见性。

## 计算

```
SoLV = (points_in_top_3 / total_grid_points) * 100
```

## 解释

|索洛夫|解读|
|------|----------------|
| 80-100% |主导的。企业拥有当地区域。 |
| 60-79% |强的。在大部分服务区都可以看到。 |
| 40-59% |缓和。覆盖范围存在显着差距。 |
| 20-39% |虚弱的。竞争对手在大部分领域占据主导地位。 |
| 0-19% |批判的。在地图结果中几乎不可见。 |

## 扩展指标

- **平均排名**：所有网格点的平均位置（较低=更好）
- **可见度分数**：加权平均值，其中前 3 = 3 分，4-10 = 1 分，10+ = 0 分
- **最差象限**：确定哪个罗盘方向排名最弱

## 格式

```

     W -------- E
  N  1  1  2  3  5  8  -

```

## 颜色映射（用于增强输出）

|职位|符号|意义|
|----------|--------|---------|
| 1 | `1` | #1 排名（最佳）|
| 2-3 | 2-3 `2`、`3` |前 3 名（强大的本地影响力）|
| 4-10 | `4`-`9` |可见但不占主导地位|
| 11-20 | `+` |埋藏在结果中|
|没有找到| `-` |暂时没有排名|

## 多关键词网格

综合分析，扫描同一格子上的2-3个关键词：

1. 主要服务关键字（例如“牙医”）
2. 品牌+地点（例如“Smith Dental Austin”）
3. 长尾意图（例如，“我附近的急诊牙医”）

**3 个关键字 7x7 扫描的成本：** 147 次 API 调用 = ~$0.29（实时）或 ~$0.088（标准）
