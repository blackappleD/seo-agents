# 关键词密度和放置规则

> 本文件是 `seo-agents` 的中文详细参考，已按当前项目能力边界适配。命令名、API 名、Schema 类型、JSON 字段和专用 SEO 术语按需保留英文。

## 来源说明

- 原始注释：Updated: 2026-04-17

基于 Semrush、Ahrefs、Yoast 发表的研究以及 Google 自己的关键字使用指南。

## 主关键词密度

**安全范围：** 总字数的 0.5% 至 2.0%。

|密度|评估|
|--------|------------|
|低于 0.5% |优化不足。可能在关键地点失踪。 |
| 0.5% - 2.0% |最佳范围。阅读自然，主题信号清晰。 |
| 2.0% - 3.0% |需要审查。可能读起来不自然。 |
| 3.0%以上 |关键词堆砌风险。可能会引发处罚。 |

**实际示例：** 对于一篇 1,000 字的文章（占 1-2%），主要关键字总共应该出现大约 10-20 次。这包括标题、正文和图像替代文本。

**回报递减：** 主要关键字的前 1-2 次提及具有最大的 SEO 权重。此后，每次额外提及都会带来更少的排名优势。优先考虑展示位置的质量而不是数量。

## 所需的放置位置

主关键字必须出现在所有这些中：

1. **title tag**（靠近前面，不埋在最后）
2. **H1标签**（靠近前面）
3. **URL slug**（小写，连字符）
4. **meta description**（自然集成）
5. **第一段/前 100 个字**（立即建立主题）
6. **至少一个图像替代文本**（强化图像搜索的主题）

## 不需要的位置

- 每个H2或H3（如果H1涵盖关键字，则副标题通过语义关系自然地携带上下文）
- 每个段落或部分（聚类看起来不自然）
- 每个内部链接的锚文本（改变锚文本）

## 次要和语义关键词

|类型 |计数|用途 |
|------|--------|--------|
|密切相关的术语 | 5-8 |通过正文内容和 H2-H6 标题分发 |
|更广泛的语义术语 | 10-15 | 10-15涵盖相关概念和用户意图变化 |
|同义词 |自然而然|提高可读性。不计入初级密度。 |

## 分配规则

- **在整个内容中均匀分布**。不要将所有提到的内容预先加载到一个部分的引言或集群中。
- **摘要中的每个部分指导：** 对于内容摘要中的每个部分，请指定：
  - 标题中应出现哪个关键字（主要或次要）
  - 章节正文是否应包含主要关键字或变体
  - 格式：“在 H2 中使用次要关键字“[term]”。正文：提及主要关键字一次。”

## 常见错误

1. **精确匹配痴迷：** 每次都使用精确的短语而不是自然变化。 Google 理解同义词和相关术语。
2. **标题堆砌：** 在每个H2和H3中放置主关键词。一次 H1 提及加上 1-2 次 H2 提及就足够了。
3. **忽略前 100 个单词：** 开头段落是标题和 H1 之后价值最高的位置。
4. **忘记图像替代文本：** 免费放置机会，也有助于图像搜索排名。
5. **将同义词计算为密度：**“Web design”、“website design”和“site design”是不同的字符串。仅将完全匹配的外观计入密度。

## 本项目适配边界

- 优先使用当前已实现的 `seo-agents` CLI 获取证据；没有 CLI 的能力作为 Agent playbook 或后续扩展处理。
- 不把外部数据源、付费 provider、浏览器渲染或地图 API 写成已执行，除非当前任务确实运行并取得证据。
- 每个 finding 必须有可复核 evidence；数据缺失时明确写“未配置”或“缺少输入”。
