## 评分系统

每个字段：**当前 + 优化 = 2 分**，**当前 = 1 分**，**缺失 = 0 分**

总分：50 分。标准化为 0-100 范围：`(score / 50) * 100`

| #|领域|积分 |优化标准 |

## 针对特定行业的权重调整

评分时，将乘数应用于对特定行业更重要的字段：

## 餐厅
- 菜单/服务链接：**x2**（对于食品相关搜索至关重要）
- 照片：**x1.5**（食物照片提高参与度）
- 预订链接：**x1.5**（预计有预订系统）
- 属性：**x1.5**（饮食、堂食/外卖/送货关键）

## 医疗保健
- 营业时间：**x1.5**（患者需要准确的营业时间）
- 属性：**x1.5**（保险、可及性、远程医疗）
- 服务清单：**x2**（保险和程序匹配）

## 法律
- 业务描述：**x1.5**（执业领域清晰度）
- 服务列表：**x2**（练习区域匹配提高可见性）
- 照片：**x0.5**（法律影响较小）

## 家庭服务
- 服务区域：**x2**（SAB 型号取决于此）
- 营业时间：**x1.5**（紧急情况下可用）
- 照片：**x1.5**（项目之前/之后的照片）

## 房地产
- 照片：**x2**（房产照片至关重要）
- 社交资料：**x1.5**（代理品牌）
- 文章：**x1.5**（列表更新）

## 汽车
- 产品：**x2**（车辆库存）
- 照片：**x2**（车辆照片）
- 服务清单：**x1.5**（销售+服务部门）

## 乘数后的重新归一化

应用行业乘数后，重新标准化，使总数保持在 0-100：

```
final_score = (weighted_raw_score / max_possible_weighted_score) * 100
```

无论哪个行业乘数处于活动状态，这都可以确保一致的评分。

## 分数解读

|分数 |评级 |行动|
|--------|--------|--------|
| 90-100 |优秀|保持发布节奏和照片新鲜度 |
| 75-89 | 75-89好 |填补补充领域的剩余空白 |
| 50-74 | 50-74需要工作|缺少重要字段，解决关键+重要差距 |
| 25-49 | 25-49差|主要的轮廓差距损害了可见性。优先考虑关键领域 |
| 0-24 |关键|profile几乎不存在或无人认领。从验证+关键字段开始 |

## 本项目适配边界

- 优先使用当前已实现的 `seo-agents` CLI 获取证据；没有 CLI 的能力作为 Agent playbook 或后续扩展处理。
- 不把外部数据源、付费 provider、浏览器渲染或地图 API 写成已执行，除非当前任务确实运行并取得证据。
- 每个 finding 必须有可复核 evidence；数据缺失时明确写“未配置”或“缺少输入”。
